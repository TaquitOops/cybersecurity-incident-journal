#  Incident Handler's Journal | Diario del Gestor de Incidentes

Este repositorio contiene mi **Diario del Gestor de Incidentes**, un proyecto desarrollado para aplicar habilidades de documentación y análisis técnico en ciberseguridad. El objetivo principal es registrar hallazgos de investigaciones y el uso de herramientas especializadas siguiendo estándares de la industria.

##  Objetivos del Proyecto
* **Documentación Profesional:** Registrar de manera precisa las fases de un incidente.
* **Aplicación de Marcos de Trabajo:** Utilizar el **Ciclo de vida de respuesta ante incidentes del NIST** (Preparación, Detección/Análisis, Contención, Erradicación y Recuperación, y Actividad posterior al incidente).
* **Análisis Forense y de Red:** Documentar la investigación de incidentes mediante la metodología de las **5 W's**.

##  Herramientas y Conceptos Aplicados
A lo largo de este diario, se documenta el uso de las siguientes herramientas y procesos de ciberseguridad:
* **Wireshark:** Análisis de tráfico de red y captura de paquetes.
* **Suricata:** Configuración de firmas y monitorización de registros IDS.
* **Splunk & Chronicle:** Gestión de eventos e información de seguridad (SIEM) para realizar consultas de telemetría.
* **Hashing:** Investigación de archivos sospechosos mediante verificación de integridad.

##  Estructura del Diario
El diario se compone de entradas críticas que demuestran competencias en:
* **Investigación de Incidentes:** Documentación detallada respondiendo a: **¿Quién?, ¿Qué?, ¿Cuándo?, ¿Dónde? y ¿Por qué?**.
* **Uso de Herramientas:** Descripción técnica de la finalidad y aplicación de software de seguridad en escenarios reales.
* **Reflexión Profesional:** Análisis sobre los desafíos enfrentados y la evolución de la comprensión en la respuesta ante incidentes.

##  Valor para el Empleador
Este documento sirve como evidencia de mi capacidad para:
* Mantener registros precisos y organizados bajo presión.
* Comunicar hallazgos técnicos de forma clara y concisa (descripciones de entre 20 y 80 palabras).
* Adoptar una mentalidad de mejora continua mediante la fase de **"Actividad posterior al incidente"** del NIST.
